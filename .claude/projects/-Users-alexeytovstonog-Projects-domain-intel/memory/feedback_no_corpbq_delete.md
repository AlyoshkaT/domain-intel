---
name: Ніколи не видаляти з corpBQ без дозволу
description: Правило — жодних DELETE/DROP у corpBQ без явного підтвердження користувача
type: feedback
---

Ніколи не виконувати DELETE, DROP, TRUNCATE або будь-яку деструктивну операцію в corpBQ (корпоративний BigQuery проєкт) без окремого підтвердження від користувача в цьому ж чаті.

**Why:** Корпоративні дані — кеш (builtwith_raw_data, similarweb_raw_data, whatcms_raw_data тощо) є спільним ресурсом. Їх випадкове видалення критичне.

**How to apply:** Перед будь-яким DELETE/DROP у corp_client() або corp_table_ref() — обов'язково показати запит і запитати явного підтвердження. Це стосується навіть "очищення" або "скидання" через UI Setup.
